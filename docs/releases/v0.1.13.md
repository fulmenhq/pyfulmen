### ğŸš€ Prepublish Workflow

- `make prepublish` now enforces a clean working tree, runs the standard
  `prepush` gates, executes `release-verify`, performs `uv run twine check
dist/*.whl dist/*.tar.gz`, and records `.artifacts/prepublish.json`.
- `release-check` refuses to proceed unless the sentinel matches the current
  `VERSION`, ensuring artifacts were built and validated before tagging.
- Helper scripts:
  - `verify_dist_contents.py` validates that wheels/sdists contain `py.typed`,
    Crucible config, schemas, and licenses.
  - `verify_local_install.py` installs the freshly built wheel using both `uv
pip` and `uv tool run pip` to guarantee compatibility.
  - `verify_published_package.py` installs from TestPyPI/PyPI and confirms the
    imported version.
  - `prepublish_sentinel.py` writes/verifies the sentinel metadata.
